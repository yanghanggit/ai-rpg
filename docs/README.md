> **写作约束（AI 编写文档时必须遵守）**
>
> - 记录**相对稳定的设计知识**，不是代码快照。
> - 禁止出现代码块（` ``` `）。
> - 以**设计意图、架构决策、跨系统调用关系**为主；代码已能自述的实现细节不必重复。
> - 文档的价值在于**宏观视角与跨系统关联**，单系统内部的步骤逻辑靠代码阅读，文档只解释"为何这样设计"和"与哪些系统交互"。
> - **符号引用原则**：核心**类名**（如 `GearItem`、`EquippedGearComponent`）相对稳定，可以行内代码引用；**变量名、函数名、字段名**随实现变化，引用它们等同于复述代码，应尽量规避。
> - **聚焦单一系统**：每篇文档只澄清本系统的设计意图；与其他系统的异同用 `→ 参见` 交叉引用表达，不在同一文档内做横向对比表。
> - **交叉引用**：涉及另一篇文档的核心概念时，用 `→ 参见：[文档标题](相对路径)` 置于段落末或独立行。
